# دليل إطلاق منصة عمرة (OMRA) بقاعدة بيانات جديدة ونظيفة

هذا الدليل يشرح لك خطوة بخطوة كيفية إنشاء قاعدة بيانات Supabase جديدة فارغة تماماً (بدون أي وكالات أو برامج تجريبية)، ورفع كود المشروع إلى GitHub، ونشره على Vercel، وتهيئة حساب المسؤول (Admin) الأول لإطلاق المنصة للعموم.

---

## 🛠️ الخطوة 1: تنظيف المشروع من مسارات التجريب (التهيؤ للإنتاج)
لمنع أي شخص من إدخال بيانات تجريبية عشوائية في الإنتاج، يُنصح بحذف كود التهيئة التجريبية:
1. اذهب للمجلد المخصص للرفع: `D:\OMRA\New folder`
2. **احذف المجلد** `src/app/api/seed` بالكامل. (حذف هذا المجلد يضمن عدم إمكانية تشغيل رابط التهيئة `https://your-domain/api/seed` على خوادم Vercel).

---

## 💾 الخطوة 2: إنشاء مشروع Supabase جديد وقاعدة البيانات
1. ادخل إلى [Supabase](https://supabase.com/) وسجل دخولك أو أنشئ حساباً جديداً.
2. اضغط على **New Project** واكتب اسماً للمشروع (مثل `OMRA-Production`)، واختر كلمة مرور قوية لقاعدة البيانات (احفظها جيداً)، واختر أقرب خادم جغرافي (مثلا غرب أوروبا أو فرنسا).
3. بعد اكتمال إنشاء المشروع (يستغرق دقيقة):
   - اذهب إلى **Project Settings (أيقونة الترس في الأسفل) > API**.
   - انسخ رابط **Project URL** (هذا هو `NEXT_PUBLIC_SUPABASE_URL`).
   - انسخ مفتاح **anon (public)** (هذا هو `NEXT_PUBLIC_SUPABASE_ANON_KEY`).
   - انسخ مفتاح **service_role (secret)** (هذا هو `SUPABASE_SERVICE_ROLE_KEY`) - *انتبه: لا تشارك هذا المفتاح أبداً*.

---

## 📐 الخطوة 3: تركيب هيكل قاعدة البيانات النظيفة (Schema)
1. في لوحة تحكم مشروع Supabase الجديد، اذهب إلى قسم **SQL Editor** من القائمة اليسارية.
2. اضغط على **New Query** (استعلام جديد).
3. انسخ محتويات ملف الهيكل الأساسي `[schema.sql](file:///d:/OMRA/src/lib/supabase/schema.sql)` بالكامل والصقها في المحرر.
4. اضغط على زر **Run** في الأسفل لتنفيذ الاستعلام وإنشاء الجداول والسياسات والأدوار.
5. افتح تبويباً جديداً في **SQL Editor**، وانسخ محتويات ملف الترقية `[migration-custom-commission.sql](file:///d:/OMRA/src/lib/supabase/migration-custom-commission.sql)` بالكامل والصقها ثم اضغط **Run** لتركيب التعديلات الأخيرة (العمولات بالبرنامج والرحلات والحدود الاستيعابية للغرف).

*الآن قاعدة بياناتك جاهزة ونظيفة 100% ولا تحتوي على أي وكالات أو برامج أو مستخدمين.*

---

## 💻 الخطوة 4: رفع المشروع إلى GitHub
سنقوم برفع محتويات المجلد النظيف `D:\OMRA\New folder` إلى حسابك الشخصي في GitHub:
1. افتح موقع [GitHub](https://github.com/) وأنشئ مستودعاً جديداً (**New Repository**).
2. اكتب اسماً للمستودع (مثال: `omra-platform`) واجعله **Private** (خاص) لحماية الكود الخاص بك، ثم اضغط **Create Repository**.
3. افتح سطر الأوامر (CMD) على جهازك واكتب الأوامر التالية بالتوالي:
   ```cmd
   # 1. الدخول لمجلد المشروع النظيف
   d:
   cd "D:\OMRA\New folder"

   # 2. تهيئة مستودع Git محلي
   git init

   # 3. إضافة جميع الملفات
   git add .

   # 4. تسجيل التغييرات
   git commit -m "Launch release: clean production setup"

   # 5. تسمية الفرع الرئيسي
   git branch -M main

   # 6. ربط المستودع المحلي بحسابك على GitHub (استبدل الرابط برابط مستودعك الفعلي)
   git remote add origin https://github.com/USERNAME/REPOSITORY-NAME.git

   # 7. رفع الملفات
   git push -u origin main
   ```

---

## 🚀 الخطوة 5: النشر على Vercel وربط البيئة
1. ادخل إلى [Vercel](https://vercel.com/) وسجل دخولك عبر حساب GitHub الخاص بك.
2. اضغط على زر **Add New > Project**.
3. ستظهر لك قائمة بمستودعاتك على GitHub، اضغط على زر **Import** بجانب مستودع `omra-platform`.
4. في صفحة الإعدادات قبل النشر، افتح تبويب **Environment Variables** وأدخل المفاتيح الثلاثة التي نسختها من Supabase في الخطوة 2:
   * الاسم: `NEXT_PUBLIC_SUPABASE_URL` | القيمة: (رابط مشروعك في Supabase)
   * الاسم: `NEXT_PUBLIC_SUPABASE_ANON_KEY` | القيمة: (مفتاح anon)
   * الاسم: `SUPABASE_SERVICE_ROLE_KEY` | القيمة: (مفتاح service_role)
5. اضغط على زر **Deploy**. سيقوم Vercel ببناء المشروع وخلال دقيقة سيعطيك رابط التطبيق النهائي المباشر على الويب!

---

## 👑 الخطوة 6: إنشاء حساب المسؤول (Admin) الأول والوحيد
بما أن قاعدة البيانات فارغة تماماً، لا يوجد أي مستخدم إداري. إليك الطريقة الآمنة لإنشاء حساب الأدمن الخاص بك:
1. اذهب إلى لوحة تحكم مشروعك في **Supabase > Authentication > Users**.
2. اضغط على زر **Add User** ثم اختر **Create User**.
3. أدخل بريدك الإلكتروني المخصص للإدارة (مثلا: `your-admin@email.com`) واكتب له كلمة مرور قوية، ثم اضغط **Save**.
4. سيظهر الحساب في القائمة؛ قم بنسخ معرف المستخدم **User ID** الخاص به (سيكون عبارة عن نص طويل مشفر مثل: `a1b2c3d4-e5f6...`).
5. اذهب إلى **SQL Editor** في Supabase وافتح تبويباً جديداً ونفذ الاستعلام التالي لتعيين صلاحيات الأدمن لهذا الحساب (استبدل المعرف بالمعرف الذي نسخته والبريد ببريدك):
   ```sql
   INSERT INTO public.profiles (id, role) 
   VALUES ('ضَع_مُعرّف_المستخدم_الذي_نسخته_هنا', 'admin');
   ```
6. اضغط **Run**.

**مبارك!** 🎉 الآن يمكنك التوجه إلى رابط موقعك المباشر والدخول إلى لوحة تحكم الإدارة `/admin/dashboard` بالبريد وكلمة المرور اللذين أنشأتهما في الخطوة 6. وستكون لوحة التحكم فارغة وجاهزة لاستقبال طلبات الوكالات الحقيقية وتفعيلها!
